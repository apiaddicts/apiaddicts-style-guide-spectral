Los scopes disponibles se definen al final del documento de definición, a nivel raíz. Se definen mediante la siguiente estructura:

Noncompliant
---------------------------------------------------------------------------------------

swagger: "2.0"
x-wso2-security:
    apim:
        x-wso2-scopes:
        -   name: read
            key: read
            roles: READ_ROLE
            description: Allows users to view records
paths:
    /pets:
        get:
            x-scope: read

Compliant
---------------------------------------------------------------------------------------

swagger: "2.0"
x-wso2-security:
    apim:
        x-wso2-scopes:
            - name: read
            key: read
            roles: READ_ROLE
            description: Allows users to view records
paths:
    /pets:
        get:
            x-scope: read
            x-aut-type: "Application & Application User"
