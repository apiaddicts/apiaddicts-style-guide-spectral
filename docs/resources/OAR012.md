Path parameter names, query parameter names, object names, and property names with more than two words must follow the configured naming convention.

Configuration Example:
---------------------------------

```yaml
rules:
  apiq:OAR012:
    given:
      - "$.paths[*][*].parameters[?(@.in == 'path' || @.in == 'query')].name"
      - "$.paths[*][*].parameters[*].schema.properties.*~"
      - "$.paths[*][*].requestBody..schema.properties.*~"
      - "$.paths[*][*].responses..schema.properties.*~"
      - "$.components.schemas.*~"
    then:
      function: apq-parameter-naming-convention
      functionOptions:
        namingConvention: "snake_case" # Default naming convention.
```

Noncompliant Example (OpenAPI 2 - camelCase convention):
---------------------------------

```yaml
swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets/{pet-id}:
    get:
      parameters:
        - in: path
          name: pet-id
          type: integer
          required: true
        - in: query
          name: PET_TYPE
          type: string
      responses:
        200:
          description: Ok
          schema:
            type: object
            properties:
              pet_name:
                type: string
```
