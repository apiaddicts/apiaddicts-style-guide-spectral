Path parameter names, query parameter names, object names, and property names with more than two words must follow the configured naming convention. You can configure snake_case, kebab-case (default), camelCase or UpperCamelCase

Configuration Example:
---------------------------------

```yaml
rules:
  apiq:OAR012:
    given:
      - "$.paths[*][*].parameters[*].name"
      - "$.paths[*][*].parameters[*].schema..properties.*"
      - "$.components.schemas..properties.*"
      - "$.components.schemas.*"
    then:
      function: apq-parameter-naming-convention
      functionOptions:
        pattern: "kebab-case" # Allowed: snake_case, kebab-case, camelCase, UpperCamelCase
```

Noncompliant Example (OpenAPI 2):
---------------------------------

```yaml
swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets/id{pet-id}:
    get:
      parameters:
        - in: path
          name: pet-id
          type: integer
          required: true
      responses:
        200:
          description: Ok
```
