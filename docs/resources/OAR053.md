Each API response must explicitly define the required response headers and must not include headers that are not allowed.
Allowing arbitrary or missing response headers can introduce security vulnerabilities and interoperability issues.

- `mandatory-headers`: Required response headers
- `allowed-headers`: Permitted response headers
- `included-response-codes`: Validated HTTP codes
- `excluded-response-codes`: Ignored HTTP codes
- `path-exclusions`: Excluded API paths


Configuration Example:
---------------------------------

```yaml
rules:
  apiq:OAR053:
    given: "$.paths[*][*].responses[*]"
    then:
      function: apq-response-headers
      functionOptions:
        mandatory-headers: "x-trace-id,x-request-id"
        allowed-headers: "idcorrelacion,x-correlacionid,x-global-trasaction-id,x-power-by,x-trace-id,x-request-id"
        included-response-codes: "*"
        excluded-response-codes: "204"
        path-exclusions: "/status,/prueba"
```

Noncompliant Example (OpenAPI 2):
---------------------------------

```yaml
swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        200:
          description: OK
```

Compliant Example (OpenAPI 2):
-------------------------------

```yaml
swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        200:
          description: OK
          headers:
            x-trace-id:
              type: string
```
